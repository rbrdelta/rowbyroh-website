Agentic Stack

How I build with AI. Infrastructure first, automation last.

AI sessions are stateless by default. Every conversation starts from zero — no memory of what was decided yesterday, no access to the notes you took last month, no awareness of your other projects. This is the system I built to fix that. Four layers, from shared state at the bottom to session roles at the top, each one solving a specific problem in the chain from "raw data scattered across platforms" to "Claude already knows what I'm working on."

L4 Session Roles make the machine meet you where you are

What
Reusable personas that prime each session with a role, behavioral expectations, and automatic context retrieval. Not chatbots — operational modes. Each one reads the vault, understands the current state, and starts from where the last session left off.
Why this layer exists
Without it, every session starts cold. You re-explain your goals, your constraints, your preferences. Session roles eliminate that tax. /pm already knows your ventures, time constraints, and decision frameworks. /designer already knows your font is IBM Plex Mono and you don't use gradients.
Design decision
Personas are slash commands (markdown files), not system prompts or fine-tunes. They're version-controlled, portable, and editable. Each one references the layers below it — they compose, not duplicate.
primes session on surface
L3 Surfaces right tool for the right mode of thinking
  • Claude Code primary
  • Claude Desktop secondary
  • Claude.ai tertiary
  • Remote Control emerging
  • Obsidian Mobile read/edit
What
Each surface serves a different cognitive mode. Code is for building — execution, iteration, shipping. Desktop is for planning — strategy, MCP tools, thinking out loud. Web is for quick lookups. Mobile is for reviewing and capturing on the go.
Key constraint
Surfaces don't share context natively. A decision made on Desktop is invisible to Code unless it flows through the vault. This is the problem the entire stack is designed to solve.
Evolution
Code is becoming the primary surface for everything, especially with remote control from mobile. The stack is consolidating around Code as the execution hub, with the vault as the shared memory layer.
reads and writes via pipelines
L2 Data Pipelines every conversation is an asset, not ephemera
  • Claude.ai sync cron 8h
  • Notion sync incremental
  • Apple Notes sync semi-auto
  • /learnings live
  • /digest live
What
Importers and sync tools that pull conversations, notes, and insights from external sources into the vault. All idempotent — re-running skips what's already imported. All incremental — only fetching what changed.
Why this matters
Without pipelines, conversations die when the session ends. With them, every insight, decision, and exploration becomes searchable, linkable, and available to future sessions. The /learnings skill captures session insights in real-time. /digest promotes the best ones into standing instructions.
Architecture
TypeScript importers share common patterns: source_id for dedup, index files for incremental tracking, dry-run mode for preview. Dev vault for testing, prod vault for real data. One pattern, multiple sources.
imports into shared state
L1 Shared State the vault is the shared brain
  • Obsidian Vault 672 notes
  • MCP Server 7 tools
  • NorthStar.md identity
  • CLAUDE.md instructions
  • Memory System per-project
  • Learnings.md cross-session
What
The foundation everything else depends on. An Obsidian vault containing every conversation, note, and document — searchable, linked, and tagged. An MCP server exposing 7 tools for creating, reading, searching, and linking notes. Configuration files that tell every surface who I am and how to work with me.
Core insight
AI sessions are stateless by default. The vault makes them stateful. When a new session starts, it doesn't ask "what are you working on?" — it reads NorthStar.md and already knows. When two surfaces need to share context, it flows through the vault, not through copy-paste.
Organization
Tags and links, not folders. Folder structure is minimal routing for imports. Classification and taxonomy emerge organically from the full body of work — not imposed top-down.
