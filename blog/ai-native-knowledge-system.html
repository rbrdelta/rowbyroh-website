672 Notes, 14 Dimensions, Zero Re-Explanation Tax

Building a knowledge system that eliminates cross-surface amnesia between AI sessions.

Every time I start a new conversation with Claude, it doesn't know what the last conversation decided. Different surfaces (web, desktop, mobile, Code) are islands. I've been building systems with Claude for months — architecture decisions, business strategy, financial models, career planning — and every session starts from a cold start.

I built a system that eliminates that tax. 672 notes from three sources, classified across 14 dimensions, connected by 2,599 auto-generated wikilinks, all searchable from inside any Claude Code session. The AI reads my entire thinking history. Zero re-explanation.

The problem: cross-surface amnesia

Claude.ai doesn't talk to Claude Code. Claude Code doesn't talk to Claude Desktop. Each surface has its own context window, its own memory, its own understanding of who I am and what I'm working on. A decision made in a Claude.ai conversation at 9pm is invisible to a Claude Code session at 9am.

Memory helps with "who is Daniel." It doesn't help with "what did Daniel decide about the taxonomy design at 2am, what were the rejected alternatives, and what file did it change."

The architecture: vault as shared brain

Import pipelines. Four sources feed the vault: Claude.ai conversations (reverse-engineered API), Notion pages (official API), Apple Notes (iOS Shortcut to iCloud Drive to Node.js importer), and Claude Code session transcripts (JSONL parser). Each pipeline is idempotent — re-running skips existing notes by source ID.

Taxonomy engine. Two-axis classification: 10 domain tags (ai, building, freight, ventures, career, design, housing, finance, meta, family) x 4 mode tags (thinking, doing, reference, creative). Plus lifecycle status (seed, growing, evergreen, archived). Scored keyword matching with title weighting (3x), length-adaptive thresholds, capped at 3 domains and 2 modes per note.

The first run over-classified badly — 63 notes hit 7+ domains. The fix: scored classification with title weighting and length-adaptive thresholds. Short notes (< 500 words) need only 1 keyword hit. Long notes (> 2000 words) need 3. Title matches count 3x because titles are intentional signals, not incidental word frequency.

Link graph. Pairwise similarity via 50% domain tag Jaccard + 50% keyword Jaccard. Title keywords weighted 3x, first 500 words of body included, stopword filtered. Each note gets a ## Related section with the top 5 most similar notes as wikilinks. Series detection finds numbered/dated conversation titles and adds prev/next navigation.

2,599 wikilinks across 503 notes. Cross-source connections work: a baby-related Apple Note links to a parenting Claude conversation. An interview prep Notion page links to a career strategy Claude.ai discussion.

MCP server. 7 tools: create, read, search, link, backlinks, daily note, screenshot. Available to Claude Code (user-scoped) and Claude Desktop. When I start a session, Claude can search the vault for prior decisions, read specific notes, and find related context — without me re-explaining anything.

The numbers

  • 672 notes: 274 Claude.ai conversations, 217 Apple Notes, 181 Notion pages
  • 14 taxonomy dimensions: 10 domain + 4 mode
  • 2,599 auto-generated wikilinks across 503 notes
  • 65 series notes with prev/next navigation (10 series detected)
  • 364 multi-domain notes (notes that bridge topics)
  • 7 MCP tools, 4 import pipelines, 13 tests, 0 duplicates

The constraint that shaped it

10-15 hours per week. Brooklyn apartment. Toddler. Day job. This isn't a full-time project — it's infrastructure built in the margins. Every design choice reflects that constraint: idempotent pipelines (safe to re-run), dry-run modes (preview before writing), dev-to-prod testing pipeline (break things in dev, not in the real vault).

The system had to be buildable incrementally. Phase 1: vault structure. Phase 2: basic MCP tools. Phase 3: AI classification. Phase 4: all sources imported, classified, linked. Each phase delivered usable value. No phase required the next one to be useful.

What it actually changed

Before: "Let me explain — I'm building a freight intelligence platform, I have 10 years of logistics experience, my composter business targets NYC's composting mandate..." every session.

After: Claude searches the vault, finds the prior conversations, reads the decisions, and starts where we left off. The re-explanation tax went from minutes to seconds. And the connections the link graph surfaces — between conversations I'd forgotten were related — create value I didn't plan for.

The principle

The goal was never "organize my notes." It was "eliminate the cost of context transfer between AI sessions." Every architectural choice followed from that: the taxonomy exists so Claude can find notes by domain, not just keyword. The links exist so related thinking surfaces automatically. The MCP server exists so the vault is accessible from where the work happens, not just where the notes live.

Infrastructure compounds. The vault I built in week one is more valuable in week six — not because I added features, but because every new conversation adds to the graph, and the graph serves every future conversation.
