Obsidian MCP Server

Bridging Claude AI with a personal knowledge vault

An MCP server that gives Claude direct read/write access to an Obsidian vault. 7 tools, 4 dependencies, idempotent pipelines, AI classification. Built to make 270+ conversations and growing actually searchable and interconnected.

The Problem

Conversations with Claude are disposable by default. Every session starts from zero. I had hundreds of conversations across Claude.ai, Claude Desktop, and Notion that contained real decisions, technical reasoning, and project context. None of it was connected. None of it was searchable from inside a new session.

The MCP protocol lets Claude call tools directly. So I built a server that turns an Obsidian vault into Claude's long-term memory.

The Tools

create_note — create notes with YAML frontmatter, tags, links
read_note — read any note by path, returns full content
search_notes — full-text search across the vault
link_notes — create bidirectional [[wikilinks]] between notes
get_backlinks — find all notes that reference a given note
process_daily_note — atomize daily notes into linked individual notes
screenshot — capture screen via PowerShell for visual verification

Claude.ai API Sync

The first pipeline problem: getting conversations out of Claude.ai and into Obsidian. Claude.ai has no export API, but the session cookie provides access to a conversation list endpoint.

The sync is incremental. It fetches the conversation index, compares timestamps against what's already in the vault, and only processes new or updated conversations. A typical sync makes one API call. Full rescan is available but rarely needed.

  • Incremental by default — compares server timestamps to local frontmatter
  • Dry-run mode for previewing changes before any writes
  • Idempotent — re-running skips existing by source_id, --overwrite to update
  • Generates YAML frontmatter with source attribution and sync metadata

Import Pipeline

Before the API sync existed, I needed to import a bulk export of 260+ conversations. The parser handles Claude.ai's JSON export format, extracting conversations into individual markdown notes with proper frontmatter.

The key design decision: idempotency via source_id. Every imported note gets a unique identifier in its frontmatter. Re-running the import skips anything that already exists. This means the pipeline is safe to run repeatedly without creating duplicates, and --overwrite lets you selectively update.

AI Classification

Notes need tags to be useful, but manually tagging 270+ notes is not realistic. The classification system uses the Anthropic API (Claude Haiku for cost efficiency) to read note content and assign tags from a controlled vocabulary.

The system has a two-tier fallback: AI classification first, heuristic matching second. If the API call fails or returns low confidence, keyword-based heuristics assign tags instead. This means classification works offline and degrades gracefully.

Tags are written to frontmatter, not inline. Obsidian indexes frontmatter tags natively, so they appear in the tag pane and are searchable without plugins.

Notion Sync

Not everything lives in Claude conversations. Project specs, meeting notes, and reference docs were in Notion. The Notion sync uses the official API with an integration token to pull pages incrementally.

Like the Claude.ai sync, it detects updated pages by comparing last_edited_time against local frontmatter. Notion's block-based content model gets flattened into clean markdown. Rich content (databases, embeds) is handled gracefully — what can be represented in markdown is, what can't is noted.

Dev/Prod Pipeline

Every change is tested in a dev vault before touching production. The dev vault is a structural mirror of prod with test fixtures. The promote pipeline runs: build, test against dev, test against prod.

npm test — runs against dev vault

npm run test:prod — runs against prod vault

npm run promote — full pipeline: build + test dev + test prod

This exists because I broke prod once by shipping a tool that worked in dev but made assumptions about vault structure that didn't hold. The two-vault system catches that class of bug.
